# タスク2：Twitch純正オーバーレイ（消えないミュート表示）を鬱陶しくなくする

> 先に `01_context.md` を読むこと。**タスク4（リロードでミュート固定）を先に直すと、この問題が自然消滅する可能性が高い** ので順番に注意。

## ゴール
動画上に Twitch 純正のオーバーレイ（特に**ミュート関連の表示）が出続けて鬱陶しい**。やること.txt の「ミュートボタンが消えない」。これを消す/出さないようにする。

## 原因（仮説・切り分けが必要）
有力なのは次の2つ。実機で**どちらが起きているか観察してから**直すこと。

1. **ミュート状態のときに Twitch が出す「クリックで音声オン」系の常駐オーバーレイ。**
   ミュートしている限り消えない種類の表示。→ **アンミュートすれば出なくなる。タスク4で非ミュート起動にすれば自然に解決**する公算が大きい。まずタスク4を済ませ、それでも残るか確認する。

2. **透明オーバーレイ（`.zap-player-overlay`）越しの操作で純正コントロールが表示され続ける。**
   `.zap-player-overlay` は iframe にフォーカスを渡さない透明 div（`01_context.md` 参照）。マウスが乗ると Twitch がコントロールバーを出し、自動で消えないように見えることがある。

## やること
### ステップ1：観察（実 Twitch ログインで）
- `/dpgk.html` を実配信で開き、**何が・どこに・いつ**出ているかを特定する。
  - ミュート中だけ出るのか、常時か。
  - マウスを動かさず放置すると消えるか（＝通常のコントロールの自動非表示か、常駐オーバーレイか）。
  - preview_screenshot や目視で、ユーザー提供スクショ（左下のアイコン等）と突き合わせる。

### ステップ2：原因別に対処
- **原因1（ミュート常駐表示）なら**：タスク4の非ミュート起動で解消するはず。未対応なら先にタスク4を実施。これで消えれば本タスクは完了でよい。
- **原因2（コントロール自動非表示の阻害）なら**：
  - `new Twitch.Player` 生成オプションで抑制できないか確認（例：`controls: false` 相当の指定が embed v1 にあるか。**公式に存在するか要確認**。無ければ次の手段へ）。
  - CSS で純正オーバーレイを隠すのはクロスオリジン iframe 内のため**直接は不可**。`.zap-player-overlay` を被せている現構成を活かし、純正UIが見えても**触れない**ようにはなっている。見た目だけ隠したい場合、プレイヤーを少し大きめに描画してコントロール帯を `.zap-player` の `overflow:hidden` で隠す等の CSS ハックも選択肢（副作用に注意、最終手段）。
  - もしくは「マウスがプレイヤー上で静止したらオーバーレイで覆って隠す」等の工夫。ただし**フォーカス遮断機能と既存のタップ＝ミュート操作を壊さないこと**。

### ステップ3：やり過ぎ防止
- 完全に純正UIをCSSで殺すのはクロスオリジン制約で困難。**現実的な落としどころ**（非ミュート起動で常駐表示を消す＋通常コントロールは自動非表示に任せる）を優先する。深追いして既存機能を壊さない。

## 完了条件
- 通常視聴時、ミュート由来の**消えないオーバーレイが出ない**（または十分に目立たない）。
- `.zap-player-overlay` のキーボードフォーカス遮断、タップ＝ミュート、自前のミュート/音量UIが従来通り機能する。
- 配信送りやチャットなど他機能に副作用がない。

## 検証
- **実 Twitch ログインが必須**（デモは映像が出ないため純正オーバーレイの再現不可）。
- ミュート/非ミュート両方、放置/マウス移動の各状況で表示を確認。
- 最終的な見え方の良し悪しは人間の判断が要るため、観察結果と対処内容をユーザーに説明し確認を依頼する。

## 注意
- コミットしない。
- クロスオリジン iframe の中身は触れない前提で、現実的な範囲に留める。
- **着手前にタスク4の状態を確認**（未対応なら先にタスク4 → 本タスクが不要になる可能性）。
